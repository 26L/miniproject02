# UI Review Checklist (Interest-based)
뉴스 요약 및 감정 분석 UI

문서 목적:  
본 체크리스트는 UI 구현물이 **사용자 관점에서 ‘관심을 가질 수 있는 UI인가’**를 기준으로  
UI Design Guideline 및 Design Token Guideline 준수 여부를 평가한다.

리뷰 방식:
- 각 항목은 **관심 있음 / 관심 없음** 중 하나로 평가한다.
- 관심 없음 항목은 반드시 개선 대상이다.
- 핵심 영역(레이아웃, 검색, 결과, 감정 표현)에 관심 없음이 존재할 경우 배포 불가.

---

## 1. 전체 구조에 대한 관심도

- [ ] Header / Content / Footer 구조가 명확해 관심 있음
- [ ] 검색 → 추천 → 결과 흐름이 직관적으로 보여 관심 있음
- [ ] 첫 화면에서 “무엇을 할 수 있는 UI인지” 즉시 이해되어 관심 있음

---

## 2. 검색 영역(Entry Point)에 대한 관심도

- [ ] 검색 입력 영역이 눈에 잘 띄어 관심 있음
- [ ] 키워드/문장 검색이 가능하다는 점이 명확해 관심 있음
- [ ] 검색 행위가 서비스의 핵심이라는 인상이 들어 관심 있음

---

## 3. 추천 뉴스 영역에 대한 관심도

- [ ] 추천 영역의 위치와 역할이 명확해 관심 있음
- [ ] “오늘 주요”와 “관심 기반” 구분이 자연스러워 관심 있음
- [ ] 추천 리스트를 눌러보고 싶다는 생각이 들어 관심 있음

---

## 4. 검색 결과 영역에 대한 관심도

- [ ] 카드 형태가 정보를 한눈에 파악하게 해 관심 있음
- [ ] 제목/요약/출처/날짜 구성이 신뢰감을 줘 관심 있음
- [ ] 여러 개의 결과를 계속 스크롤해 보고 싶어 관심 있음

---

## 5. 감정 분석 표현에 대한 관심도

- [ ] 감정 상태가 직관적으로 이해되어 관심 있음
- [ ] 색상과 텍스트 조합이 자연스러워 관심 있음
- [ ] 감정 정보가 뉴스 이해에 실제로 도움이 되어 관심 있음

---

## 6. 색상 사용에 대한 관심도

- [ ] 전체 색상 톤이 서비스 성격과 잘 맞아 관심 있음
- [ ] 주요 액션(검색 등)이 색상으로 잘 드러나 관심 있음
- [ ] 색상 사용이 과하지 않고 안정적이라 관심 있음

---

## 7. 컴포넌트 일관성에 대한 관심도

- [ ] 카드/버튼/리스트 스타일이 통일되어 관심 있음
- [ ] 화면 내에서 “어색한 UI”가 느껴지지 않아 관심 있음
- [ ] 다른 화면이 추가되어도 같은 규칙이 적용될 것 같아 관심 있음

---

## 8. 인터랙션 & 반응에 대한 관심도

- [ ] Hover/클릭 반응이 자연스러워 관심 있음
- [ ] 애니메이션이 방해되지 않고 적절해 관심 있음
- [ ] UI가 “살아 있다”는 느낌을 주어 관심 있음

---

## 9. 반응형 UI에 대한 관심도

- [ ] Desktop 화면이 안정적이고 정보 파악이 쉬워 관심 있음
- [ ] Mobile 환경에서도 핵심 기능이 유지될 것 같아 관심 있음
- [ ] 추천 영역이 모바일에서도 접근 가능할 것 같아 관심 있음

---

## 10. 종합 관심도 평가

- 전체적으로 이 UI는 **계속 사용해 보고 싶은가**
  - [ ] 관심 있음
  - [ ] 관심 없음
