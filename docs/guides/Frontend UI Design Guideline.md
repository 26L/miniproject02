# Frontend UI Design Guideline
뉴스 요약 및 감정 분석 UI

문서 목적: 뉴스 요약 및 감정 분석 API와 연동되는 프론트엔드 UI의 **레이아웃/컴포넌트/색상/상태 표현 규칙**을 표준화한다.

---

## 1. 목적 (Purpose)

- 검색 → 추천 → 결과 확인의 핵심 사용자 흐름을 **단일 화면에서 일관되게 제공**
- 컴포넌트 간 **시각적 일관성** 확보
- 색상/타이포/상태 표현을 규정해 **품질 편차를 제거**

---

## 2. 화면 구조 (Layout)

### 2.1 전체 레이아웃

- 구성: **Header / Content / Footer**
- Content는 **중앙 정렬**을 기본으로 한다.
- Content는 **좌·우 2컬럼**으로 구성한다.

구조(논리):

Header  
Search Bar (Full Width)  
Content  
- Left Column: 추천 뉴스  
- Right Column: 검색 결과  
Footer  

---

## 3. 기능별 UI 배치 규칙

### 3.1 뉴스 검색 영역

- 기능: 뉴스 **키워드 또는 문장** 검색
- 위치: **Header 바로 아래**, **가로 전체 폭**
- 규칙:
  - 입력창은 사용자가 즉시 인지할 수 있어야 한다.
  - 검색 실행 시 결과 영역이 갱신되어야 한다.

---

### 3.2 뉴스 추천 영역 (Left Column)

- 기능:
  - 오늘의 주요 뉴스 노출
  - 사용자 관심 기반 추천 노출
- 위치: **Content 왼쪽 컬럼**
- 표현: **리스트(List)**
- 규칙:
  - 리스트는 스크롤 가능해야 한다(아이템이 많을 경우).
  - 아이템 클릭 시 검색 결과 영역과 **연동**될 수 있어야 한다(동일 기사 표시 또는 검색 조건 반영).

---

### 3.3 뉴스 검색 결과 영역 (Right Column)

- 기능:
  - 검색 결과 노출
  - 요약 및 감정 분석 결과 표시
- 위치: **Content 오른쪽 컬럼**
- 표현: **카드(Card)**
- 필수 정보(각 카드):
  - 제목
  - 요약
  - 감정 분석 결과
  - 출처
  - 날짜(발행일 또는 수집일)

---

## 4. 색상 시스템 (Color System)

### 4.1 기본 원칙

- 색상 컨셉: **Ocean Blue**
- 본 문서는 색상의 **역할(role)**만 정의한다.
- 색상 값(hex)은 별도 **Design Token 문서**를 단일 기준으로 한다.

### 4.2 컬러 역할 정의

| Role | 사용처 기준 |
|---|---|
| Primary | 주요 CTA(검색 실행), 핵심 강조, 링크/포커스 |
| Secondary | 섹션 헤더, 활성 상태, 보조 강조 |
| Accent | 배지/태그, 부분 하이라이트 |
| Success | 긍정 상태(Positive) |
| Warning | 중립/주의(Neutral) |
| Error | 부정/오류(Negative/Error) |
| Surface | 기본 배경(페이지/패널) |
| Text | 기본 텍스트(본문/제목) |

---

## 5. 감정 분석 표현 규칙 (Emotion)

### 5.1 표현 방식

- 감정은 **텍스트 + 색상**으로 동시 표현한다.
- 색상만으로 의미가 전달되도록 의존하지 않는다(접근성).

### 5.2 감정 상태 정의

- Positive → Success
- Neutral → Warning
- Negative → Error

---

## 6. 컴포넌트 스타일 원칙 (Component Style)

### 6.1 공통 원칙

- 모서리: 부드러운 라운드 스타일 적용
- 그림자: 과하지 않은 Soft Shadow 사용
- 상태 변화: Hover/Active/Focus는 사용자 피드백을 제공해야 한다.

### 6.2 일관성 규칙

- 동일한 역할의 컴포넌트는 다음을 동일하게 유지한다:
  - 형태(레이아웃/간격)
  - 색상 규칙(역할 기반)
  - 인터랙션(hover/active/focus)

---

## 7. 타이포그래피 (Typography)

- 기본 폰트:
  - 영문: Poppins
  - 한글: Noto Sans KR (Google Fonts)
- 위계:
  - 제목 / 본문 / 메타(출처·날짜)는 크기·두께·색 대비로 구분한다.
- 가독성:
  - 본문은 정보 가독성을 최우선으로 한다.

---

## 8. 인터랙션 & 애니메이션 (Interaction)

- 목적: 상태 변화와 화면 전환을 명확히 전달한다.
- 규칙:
  - 과도한 애니메이션 금지
  - 전환은 짧고 일관된 패턴(Slide/Fade) 사용
  - 접근성: reduced motion 환경을 고려한다.

---

## 9. 반응형 원칙 (Responsive)

- 기준: **Desktop First**
- Desktop:
  - 2컬럼 유지(추천/결과 동시 노출)
- Mobile/Tablet:
  - 단일 컬럼으로 전환
  - 추천 영역은 상단 배치 또는 별도 접근 방식(메뉴/드로어) 허용

---

## 10. 준수 원칙 (Compliance)

- 본 문서는 UI 구현의 **기준 문서**다.
- 신규 기능/화면은 본 가이드라인을 기본으로 한다.
- 예외는 반드시 사유와 범위를 문서화한다.
