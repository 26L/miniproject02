# 📏 Development Assessment & Reporting Guide

이 문서는 **News Insight Pro** 프로젝트의 품질을 평가하는 기준과, 추후 프로젝트 결과 보고서(포트폴리오)를 작성하기 위한 가이드라인을 제공합니다.

---

## 1. 📊 진척률 및 완성도 평가 모델 (Progress & Maturity Model)

프로젝트의 현재 상태를 객관적인 수치와 단계로 정의하여 관리합니다.

### A. 정량적 진척률 (Quantitative Progress)
*각 모듈의 난이도와 중요도에 따라 **가중치(Weight)**를 부여하여 계산합니다.*

| 단계 | 세부 모듈 | 가중치(%) | 완료 여부 | 비고 |
| :--- | :--- | :---: | :---: | :--- |
| **Phase 1** | **환경 설정 및 설계** | **10%** | | 폴더구조, Git, API키, 문서화 |
| **Phase 2** | **Backend 기본 (DB/API)** | **15%** | | FastAPI 구동, DB 스키마, CRUD |
| **Phase 3** | **Core Logic (수집/AI)** | **25%** | | **[핵심]** 크롤러 및 GPT 연동 |
| **Phase 4** | **Frontend UI 구축** | **20%** | | React 세팅, 컴포넌트 퍼블리싱 |
| **Phase 5** | **통합 연동 (Integration)** | **20%** | | API 연결, 비동기 상태 관리 |
| **Phase 6** | **안정화 및 배포 (Polish)** | **10%** | | 예외 처리, 문서 정리 |
| **Total** | **전체 공정** | **100%** | **0%** | *(현재 달성률)* |

> **📈 진척률 계산 공식:** $\sum (\text{완료된 항목의 가중치})$

### B. 서비스 성숙도 단계 (Service Maturity Levels)
*소프트웨어 생명주기(SDLC)에 따른 현재 완성도 단계를 정의합니다.*

* **Level 0: Setup (초기 설정)**
    * [ ] 개발 환경 구축 완료, "Hello World" 서버 구동 확인.
* **Level 1: PoC (개념 증명)**
    * [ ] UI 없이 Swagger(백엔드) 상에서 뉴스 수집 및 AI 요약 기능이 정상 작동함.
* **Level 2: MVP (최소 기능 제품)**
    * [ ] Frontend와 Backend가 연동되어, 사용자가 검색부터 결과 확인까지 멈춤 없이 수행 가능함. (디자인은 미흡해도 기능은 동작)
* **Level 3: Beta (기능 개선)**
    * [ ] 디자인(Tailwind CSS)이 적용되고, 감성 분석 시각화(색상)가 구현됨.
    * [ ] 로딩 스피너, 에러 메시지 등 UX 요소 적용 완료.
* **Level 4: Production Ready (배포 가능)**
    * [ ] 모든 콘솔 에러 제거, 코드 리팩토링 완료.
    * [ ] `README.md` 및 결과 보고서 작성 완료.

## 2. ✅ 자체 평가 체크리스트 (Self-Assessment)

개발 완료 후(Level 4 도달 시) 아래 항목들을 점검하여 프로젝트의 품질을 최종 보증합니다.

### A. 기능적 요구사항 (Functionality)
- [ ] **데이터 수집:** 검색 키워드에 맞는 뉴스가 정확히 크롤링 되는가? (중복 제거 확인)
- [ ] **AI 분석:** 기사 요약이 3문장 이내로 깔끔하게 떨어지는가? / 감성 분석 결과(긍정/부정)가 내용과 일치하는가?
- [ ] **UI 연동:** 검색 -> 로딩 -> 결과 출력의 과정이 끊김 없이 매끄러운가?
- [ ] **예외 처리:**
    - 검색 결과가 없을 때 안내 메시지가 뜨는가?
    - API 키 오류나 네트워크 연결 끊김 시 앱이 멈추지 않고 에러를 표시하는가?

### B. 코드 품질 (Code Quality)
- [ ] **Backend:**
    - 모든 I/O 작업(DB, API)에 `async/await`가 올바르게 적용되었는가?
    - Pydantic 스키마를 통해 데이터 타입 검증이 엄격하게 이루어지는가?
    - `.env` 파일을 사용하여 민감 정보가 분리되었는가?
- [ ] **Frontend:**
    - 컴포넌트가 기능별로 적절히 분리되었는가? (하나의 파일이 너무 길지 않음)
    - `any` 타입을 남발하지 않고 TypeScript Interface를 정의했는가?
    - 콘솔에 붉은색 에러 로그나 불필요한 `console.log`가 남아있지 않은가?

## 3. 📝 결과 보고서 작성 가이드 (Report Template)

프로젝트 완료 후, 아래 목차에 따라 정리하면 훌륭한 **기술 포트폴리오**가 됩니다.

### [1] 프로젝트 개요 (Introduction)
* **프로젝트명:** News Insight Pro
* **한 줄 소개:** 실시간 뉴스 크롤링 및 GPT 기반 감성 분석 대시보드
* **개발 동기:** 수많은 뉴스 데이터 속에서 긍정/부정 여부와 핵심 내용을 빠르게 파악하기 위함.
* **최종 달성:** 진척률 100%, 성숙도 Level 4 달성.

### [2] 시스템 아키텍처 (Architecture)
* (여기에 `PROJECT_CONTEXT.md`에 있는 Mermaid 다이어그램이나 구조도를 삽입)
* **Backend:** FastAPI, SQLite, OpenAI API (Async 처리 강조)
* **Frontend:** React, Tailwind CSS, React Query (상태 관리 강조)

### [3] 핵심 트러블 슈팅 (Troubleshooting) - *가장 중요*
*단순한 기능 나열보다, 개발 중 겪은 문제와 해결 과정을 적는 것이 좋습니다.*

* **문제 상황:** (예: 뉴스 10개를 순차적으로 분석하니 시간이 너무 오래 걸림)
* **원인 분석:** (예: 동기식 `requests` 라이브러리 사용으로 인한 블로킹 발생)
* **해결 과정:** (예: `httpx`와 `asyncio.gather`를 도입하여 병렬 비동기 처리로 변경)
* **결과:** (예: 처리 속도 10초 -> 2초로 80% 단축)

### [4] 주요 기능 시연 (Features)
* **뉴스 검색:** (스크린샷 첨부)
* **AI 상세 분석:** (요약 및 감성 점수 화면 스크린샷 첨부)
* **반응형 UI:** (모바일 화면 예시)

### [5] 회고 및 배운 점 (Retrospective)
* FastAPI의 비동기 처리에 대한 깊은 이해.
* 프롬프트 엔지니어링을 통해 AI 응답 형식을 제어하는 노하우 습득.
* React와 Backend 간의 CORS 이슈 해결 경험.

## 4. 💾 커밋 메시지 컨벤션 (Git Commit Guide)

일관성 있는 커밋 메시지는 협업과 이력 관리에 필수입니다.

* `feat`: 새로운 기능 추가 (예: `feat: 뉴스 검색 API 구현`)
* `fix`: 버그 수정 (예: `fix: CORS 에러 해결`)
* `docs`: 문서 수정 (예: `docs: README.md 업데이트`)
* `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
* `refactor`: 코드 리팩토링 (기능 변경 없음)
* `chore`: 빌드 업무, 패키지 매니저 설정 등
